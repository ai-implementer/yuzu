# 全文検索

ヘッダーの検索ボックス（<kbd>/</kbd> または <kbd>Cmd</kbd>+<kbd>K</kbd> で
フォーカス）から日本語で検索できます。エンジンは分かち書き（vaporetto）＋
自前 BM25 で、インデックスは静的ファイル（`dist/_search/`）です。
**サーバは不要**で、CDN や GitHub Pages のような静的ホスティングだけで
動きます。

## セクション単位のヒット

検索は**セクション（h2 / h3）単位**です。結果は「ページ › 見出し」の形で並び、
選ぶと `#アンカー` へ直接ジャンプします。抜粋はクエリ一致箇所の周辺を
動的に生成し、分かち書き単位でハイライトします。

## タイポトレランス

1 文字の誤字（編集距離 1）に寛容です。たとえば「ダーくモード」で検索しても
「ダークモード」がヒットします。

## フレーズ検索

`"..."` で囲むと**フレーズ検索**になり、囲んだ語句が連続で出現する
セクションだけに絞り込めます（例: `"ライブリロード"`）。

- 引用部は誤字補正・同義語展開の対象外の**完全一致**です
- 全角引用符・カーリー引用符でも囲めます。閉じ忘れは末尾までが引用部です
- 抜粋ではフレーズ全体が 1 まとまりでハイライトされます

引用符なしの複数語クエリにも**近接ブースト**が働き、語がクエリ順に隣接して
出現するページが上位に来ます（ヒット集合は変わらず、順位だけが変わります）。

## 同義語とゆれ表記の展開

`lint.terms` の用語辞書と `search.synonyms` の同義語グループは、検索の
**クエリ拡張**に使われます。ゆれ表記で検索しても正表記のページがヒットし、
ハイライトも正表記側に付きます。

このサイトの設定は次のとおりです。試しに `サーバー`（長音符あり）や
`ダイアグラム` で検索すると、`サーバ` や `図` と書かれたページが
ヒットします:

```jsonc
"lint": {
  "terms": { "サーバ": ["サーバー"], "ユーザ": ["ユーザー"] }
},
"search": {
  "synonyms": [["図", "ダイアグラム"], ["設定", "コンフィグ"]]
}
```

## コードブロックの検索（indexCode）

`search.indexCode: true` にすると**フェンスコードブロックの中身も検索対象**に
なり、関数名や設定キーで設計書を引けます（既定は off）。このサイトでは
有効にしているので、たとえば `maxTermsPerShard` のような設定キーでも
このドキュメントを検索できます。

mermaid / openapi / jsonschema / math のような特別レンダリングされる
ブロックのソースは対象外です。

## ターミナルから検索する

`yuzu search` は**ブラウザとまったく同じエンジン**（同一のトークナイザと
インデックス）で検索します:

```bash
yuzu search "検索したい言葉"
yuzu search '"ライブリロード"'   # フレーズ検索も同じ構文
```

## 再訪問を速くする OPFS キャッシュ

ブラウザは初回検索時にインデックス（辞書・シャード・分かち書きモデル）を
ダウンロードし、対応環境では **OPFS**（Origin Private File System）へ
保存します。再訪問時は `manifest.json` の `contentHash` を照合し、一致すれば
再フェッチせずローカルから読みます。OPFS 非対応・非セキュアコンテキストでは
従来どおりのフェッチのみの経路へ自動フォールバックします。

> [!NOTE]
> 検索 UI の動作確認は `yuzu preview` / `yuzu dev` 経由で行ってください。
> `file://` で開くと fetch が動かず、OPFS もセキュアコンテキスト
> （https か localhost）を要求します。`search.enabled: false` で
> 機能ごと無効化できます。
