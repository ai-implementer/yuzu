# 執筆の基本

## ページとナビゲーション

`content/**/*.md` の 1 ファイルが 1 ページになり、**ディレクトリ階層が
そのままサイドバーの階層**になります。`guide/writing.md` は
`/guide/writing/` として配信されます。

- 並び順は frontmatter の `order` 昇順（未指定はファイル名順で最後尾）
- ディレクトリの表示名・並び順・リンク先は、その配下の `index.md` から取ります
- h2 / h3 見出しは右側の「目次」に自動で載ります
- 全ページはサイドバーの表示順で連結され、ページ末尾に**前後ページリンク**、
  ページ上部に**階層パンくず**が付きます

## frontmatter

各ページの先頭に YAML frontmatter を書けます。使えるキーは次の 6 つです
（未知のキーは `yuzu lint` が警告します）:

```yaml
---
title: ページタイトル # ナビの表示名（未指定は h1 → ファイル名）
order: 1 # ナビの並び順（未指定はファイル名順で最後尾）
draft: true # ビルドから除外する
description: 説明 # meta description
llms: false # llms.txt / llms-full.txt から除外する
aliases: ["guide/old-name/"] # 旧 URL（リダイレクトを生成）
---
```

### 下書き（draft）

`draft: true` のページは通常ビルドから除外されます。書きかけの確認には
`yuzu dev --drafts` / `yuzu build --drafts` を使うと、**下書きバナー付き**で
プレビューできます（通常ビルドに戻すと下書きの出力は自動で掃除されます）。

### ページの移動とリダイレクト（aliases）

ページを移動・改名したら、旧 URL を `aliases` に残します。ビルド時に
旧 URL へ**リダイレクト HTML**（meta refresh + canonical + JS フォールバック）が
生成され、外部からのブックマーク・リンクが切れません:

```yaml
---
title: 品質チェック
aliases: ["guide/lint/"] # 以前この URL だった
---
```

- パスはサイト内の route 形式で書きます（先頭 `/` と末尾スラッシュの省略は可）。
  `baseUrl` 設定にも自動で追随します
- エイリアスが実ページや他のエイリアスと衝突すると、ビルドは中断し
  `yuzu check` がエラーとして報告します
- エイリアスを外すと、旧リダイレクトは次のビルドで自動掃除されます
- リダイレクトページは検索インデックス・llms.txt に載りません。また
  **本文からエイリアス URL へリンクすると `yuzu check` がリンク切れとして
  指摘します**（サイト内のリンクは常に移動後の URL へ張る運用です）

## GFM 記法

GitHub Flavored Markdown の表・タスクリスト・打ち消し線・autolink が使えます。

| 記法 | 例 |
| --- | --- |
| 表 | この表がそれです |
| タスクリスト | 下の例 |
| 打ち消し線 | ~~古い記述~~ |
| autolink | <https://github.com/ai-implementer/yuzu> |

- [x] 構成を決める
- [ ] 本文を書く

## Admonition

GitHub 互換の `> [!NOTE]` 形式で注意書きが書けます（5 種類）。

> [!NOTE]
> 補足情報。ラベルは既定で英語です。

> [!TIP]
> 知っていると便利な使い方のヒント。

> [!IMPORTANT]
> 目的を達成するために必ず知っておくべきこと。

> [!WARNING]
> 問題を避けるために注意が必要な内容。

> [!CAUTION] 破壊的操作に注意
> `> [!CAUTION] タイトル` のように 1 行目へ書くと、ラベルを日本語などへ
> 上書きできます。

## 脚注

本文に脚注参照を書けます[^sample]。表示では定義がページ末尾に集約され、
相互リンクされます。

[^sample]:
    脚注の本文。`yuzu fmt` は定義をこの位置のまま温存し、未参照の定義も
    消しません。

## 画像と添付ファイル

置き場所は 2 通りあります:

1. **`public/` に置く** — そのまま `dist/` へコピーされ、`/images/...` の
   サイト絶対パスで参照します（サイト共通のロゴなど向け）
2. **ページと同じディレクトリに置く** — `![図](diagram.png)` のように
   相対パスで参照します。content 配下の `.md` 以外のファイルは `dist/` へ
   自動コピーされ、リンクは正しい URL に解決されます（ページ専用の図向け）

![yuzu ロゴ](/images/yuzu-logo.svg)

参照切れはどちらの方式でも `yuzu check` が検出します。

## その他の記法

引用・キーボード表記・区切り線にもテーマのスタイルが当たります。

> 引用ブロック。出典の明示や補足に使います。

検索ボックスへは <kbd>/</kbd> または <kbd>Cmd</kbd>+<kbd>K</kbd> で
フォーカスできます。
